# Кейс №2: IoT-шеринг самокатов «Вжух» (Интеграция Предсказательного ИИ)

Кейс демонстрирует проектирование высоконагруженной IoT-системы шеринга самокатов с интеграцией ИИ-агента для прогнозирования спроса, динамического ценообразования и управления геозонами.

---

## 🤖 1. Роль ИИ-Агента в экосистеме (AI Smart Mobility)
В отличие от классических систем, бэкенд «Вжух» управляется ИИ-моделью класса Predictive Analytics:

*   **Динамический «Тариф для ленивых»:** ИИ анализирует плотность самокатов в радиусе 500 метров, текущую погоду, заторы на дорогах и автоматически рассчитывает коэффициент наценки для парковки вне зон.
*   **Оптимизация баланса сети (Rebalancing):** ИИ прогнозирует дефицит самокатов у станций метро за 40 минут до начала часа пик и формирует автоматические задачи для полевых курьеров-скаутов на перемещение техники.

---

## 🔌 2. Интеграционный контракт ИИ-телеметрии (JSON Specs)

Каждую секунду IoT-модуль самоката отправляет на брокер сообщений пакет данных, который мгновенно подтягивается ИИ-агентом для пересчета матрицы спроса:

```json
{
  "scooter_id": "vzhuh-iot-77492",
  "timestamp": "2026-07-01T21:15:00Z",
  "telemetry": {
    "gps": { "latitude": 59.9342, "longitude": 30.3350, "speed_kmh": 15.2 },
    "hardware": { "battery_percentage": 88, "lock_state": "unlocked" }
  },
  "ai_routing": {
    "current_zone_type": "standard_parking",
    "predicted_demand_factor": 1.45,
    "nearest_high_demand_cluster_id": 402
  }
}
```

---

## 📊 3. Системные и Нефункциональные требования (NFR)
*   **NFR-PERF-01 (Нагрузка):** Поддержка 3x пиковой нагрузки в часы пик и выходные.
*   **NFR-LATENCY:** Время отклика ИИ-модуля при расчете динамического тарифа в момент нажатия кнопки «Забронировать» не должно превышать **350 мс**.
*   **NFR-NET:** Обновление GPS-телеметрии 1 раз в секунду по протоколу **MQTT** для экономии трафика IoT-модулей.
*   **NFR-SEC:** Полный запрет на хранение номеров карт. Токенизация по стандарту PCI DSS.
