# Кейс: Коворкинг — Тест-кейсы (негативные сценарии)

## Список тестовых сценариев

| № | Сценарий | Ожидаемое поведение |
|---|----------|---------------------|
| 1 | Два сотрудника пытаются забронировать одно место на одну дату одновременно | Первый получает бронь, второй — ошибку 409 Conflict с сообщением: "Это место уже забронировано на выбранную дату" |
| 2 | Сотрудник пытается забронировать место на дату позже чем через 2 недели | Ошибка 400 Bad Request: "Бронирование возможно не более чем на 2 недели вперёд" |
| 3 | Сотрудник пытается отменить бронь, которая уже была отменена | Ошибка 404 Not Found: "Бронь не найдена" или статус уже "cancelled" |
| 4 | Сотрудник пытается перенести бронь на дату, когда место уже занято | Ошибка 409 Conflict: "Новое место уже забронировано на эту дату" |
| 5 | Администратор блокирует место, на которое есть активные брони в будущем | Система предупреждает о бронях, предлагает отменить или перенести |
| 6 | Система автоматически отменяет брони при превышении порога 5% неиспользованных | Статус броней меняется на "cancelled", администратор получает уведомление |
| 7 | Сотрудник пытается забронировать место в нерабочий день (если есть такие правила) | Ошибка валидации: "Выбранная дата не является рабочим днём" |
| 8 | Сотрудник пытается забронировать место без авторизации | Ошибка 401 Unauthorized: "Требуется авторизация" |
| 9 | Администратор пытается удалить место, на которое есть активные брони (статус "active") | Ошибка: "Нельзя удалить место с активными бронями. Сначала отмените или перенесите брони" |
| 10 | Невалидный JSON в запросе POST /bookings | Ошибка 400 Bad Request: "Невалидный формат запроса" |

---

## Критерии приемки (Acceptance Criteria)

### Сценарий: Успешное бронирование
- [ ] Место отображается как "свободно" до бронирования
- [ ] После бронирования статус места меняется на "забронировано"
- [ ] Сотрудник видит подтверждение с датой и номером места
- [ ] Бронь появляется в списке активных броней сотрудника

### Сценарий: Отмена бронирования
- [ ] Бронь меняет статус на "cancelled"
- [ ] Место снова отображается как "свободно" на эту дату
- [ ] Сотрудник получает уведомление об отмене
- [ ] Бронь не удаляется из базы (сохраняется история)

### Сценарий: Конфликт бронирования
- [ ] Система возвращает 409 Conflict
- [ ] Сообщение понятно объясняет причину
- [ ] Сотрудник может выбрать другое место или дату
- [ ] Бронирование не создаётся

### Сценарий: Горизонт бронирования
- [ ] Система возвращает 400 Bad Request
- [ ] Сообщение: "Бронирование возможно не более чем на 2 недели вперёд"
- [ ] Бронирование не создаётся

### Сценарий: Администрирование мест
- [ ] Администратор может добавлять, редактировать, блокировать места
- [ ] Блокировка места не влияет на уже существующие брони (но предупреждение есть)
- [ ] Статистика по бронированиям обновляется в реальном времени
